# 任务列表修复与优化方案

针对您提出的三个体验问题，我制定了以下统一的修复与改进计划：

## 1. 历史记录中无法修改任务的完成状态
**问题分析**：目前在历史记录双击打开的“任务详情 (TaskDetailWindow)”中，右下角的完成状态徽章（显示 ⏳ 或 ✓ 的区域）仅作展示，无法点击交互。
**解决方案**：
- 在 `TaskDetailWindow.xaml` 中，为状态徽章增加交互反馈（`Cursor="Hand"`）和鼠标点击事件。
- 在后台代码 `TaskDetailWindow.xaml.cs` 中实现状态切换逻辑：点击后在“未完成”与“已完成”之间翻转，并同步更新完成时间。点击“保存修改”后即可生效到数据库。

## 2. 新增任务支持 Tab 键快捷切换优先级
**问题分析**：在输入框打字时，如果想顺手改优先级，必须使用鼠标去点，降低了录入效率。
**解决方案**：
- 在 `MainWindow.xaml.cs` 的 `TaskInput_KeyDown` 输入框键盘事件中拦截 `Tab` 键。
- 触发 Tab 键时，阻止默认的控件焦点切换（`e.Handled = true`），并让绑定的 `NewTaskPriority` 在 `0(无) -> 1(中) -> 2(高) -> 3(紧急)` 之间进行循环切换，实现全键盘录入。

## 3. 历史记录日期控件 (DatePicker) 丑陋问题
**问题分析**：WPF 默认的 `DatePicker` 控件保留了传统 Windows 的灰白边框与底色，在 Taskato 现有的深色圆角 UI 下显得极其突兀。
**解决方案**：
- 在 `App.xaml` 中编写一套简约暗色的 `DarkDatePicker` 样式。
- 覆盖其默认的边框和背景，统一使用与其他输入框相同的半透明暗色背景（`#0FFFFFFF`）、微弱的边框描边以及统一的白色字体。
- 将 `HistoryWindow.xaml` 中的开始日期与结束日期控件应用该样式，使其完全融入暗黑主题。
