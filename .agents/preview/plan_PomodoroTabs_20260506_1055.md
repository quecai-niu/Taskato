# 多番茄钟选项卡 (Tab/Carousel) 界面改造计划

## 1. 当前问题分析
目前多组番茄钟以垂直列表的形式堆叠在主番茄钟下方，随着数量增加会导致主页显得非常拥挤凌乱，且视觉焦点分散，不符合简约精致的设计理念。

## 2. 改造方案：选项卡式切换 (Tabbed UI)
将多番茄钟显示模式从“垂直平铺”改为“选项卡切换”：
- **UI 呈现**：在番茄钟模块顶部增加一个胶囊形状的选项卡栏 (Pill Tab Bar)。
- **交互逻辑**：主页永远只显示**一个**番茄钟的完整表盘和操作按钮。用户点击顶部的选项卡，可以在“主番茄钟”、“附加番茄钟1”、“附加番茄钟2”之间无缝切换显示内容。
- **视觉美学**：采用 Glassmorphism（玻璃拟物化）风格，带平滑滑动动画的选中指示器，彻底脱离原有的列表感。

## 3. 具体执行步骤

- [ ] **[UI]** 移除 `MainWindow.xaml` 中原有的附加计时器 `ItemsControl` 垂直列表区域。
- [ ] **[Logic]** 在 `MainViewModel` 中新增 `CurrentActiveTimer` 属性，用于绑定当前正在显示的番茄钟实例（主番茄钟或附加番茄钟）。
- [ ] **[UI]** 在 `MainWindow.xaml` 的番茄钟模块顶部添加由 `RadioButton` 构成的水平选项卡区域，绑定到 `AdditionalTimers` 和主番茄钟的集合。
- [ ] **[Logic]** 更新控制按钮的 Command 绑定，使其作用于当前的 `CurrentActiveTimer` 而不仅仅是硬编码的主番茄钟。
