# 首页显示未完成历史任务的重构方案

目前首页只显示“今日创建”的任务，且任务卡片上的时间也仅显示“时:分”。为了让过去未完成的任务顺延显示在首页，同时解决日期显示不兼容的问题，我制定了以下重构方案：

## 1. 核心数据查询调整 (DatabaseService.cs)
将 `GetTodayTasksAsync` 的查询逻辑修改为：
- 获取条件：**“今天创建的任务 (无论是否完成) + 过去未完成的任务 (IsCompleted == false)”**
- 排序规则保持不变（按优先级 -> 拖拽排序 -> 创建时间排列）。

## 2. 任务卡片时间显示适配 (Converters.cs)
修改 `TaskTimeDisplayConverter`，让它能智能判断并显示日期：
- 如果是**今天**创建的：显示 `14:30 创建`
- 如果是**昨天**创建的：显示 `昨天 14:30 创建`
- 如果是**更早**创建的：显示 `5月5日 14:30 创建`
- （完成时间的显示逻辑同理，如果是以前完成的也会带上日期）。

## 3. 首页头部标题调整 (MainWindow.xaml.cs)
目前顶部的文本是写死的 `今日 · 2026年x月x日`，容易让人误以为列表里**只有**今天的任务。
- **调整建议**：将文本修改为 `待办与今日 · 2026年x月x日` 或 `我的待办 · 5月7日`。
- **动态刷新**：增加一个简单的跨天刷新机制，或者在 `LoadTodayTasksAsync` 被调用时顺便更新一下顶部的日期文本，避免挂机跨天后顶部日期依然停留在昨天。

## 4. UI 细节优化 (MainWindow.xaml)
为跨天延期的任务在 UI 上增加一点视觉区分（可选）：
- 比如在时间旁边加一个微小的红色标记或警告色，提示这是“逾期”顺延下来的任务。*(根据您的极简需求，这部分可以保持原样，仅通过上述的“昨天 14:30”文字来体现即可)*。
