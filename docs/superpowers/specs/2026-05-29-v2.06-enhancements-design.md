## 1. 跨天日期不更新（Bug 修复）

### 现状

MainWindow 构造函数中硬编码 `DateLabel.Text = $"待办与今日 · {DateTime.Today:M月d日}"`，只执行一次。任务列表 `LoadTodayTasksAsync()` 只在增删改操作后触发，无定时刷新。

### 方案

MainViewModel 新增 `DispatcherTimer`，每 30 秒 tick：

1. 比较 `_currentDate` 与 `DateTime.Today`
2. 若变化：更新 `DateLabelText` 属性 → 调用 `LoadTodayTasksAsync()`

### 改动清单

| 文件 | 改动 |
|---|---|
| `MainViewModel.cs` | 新增 `DateLabelText` 属性、`DispatcherTimer`、日期检测逻辑 |
| `MainWindow.xaml` | DateLabel Text 改为 `{Binding DateLabelText}` |
| `MainWindow.xaml.cs` | 删除 `DateLabel.Text = ...` 硬编码行 |

## 2. 历史任务多条件筛选排序

### 现状

HistoryWindow 只有日期范围 + 关键词搜索，排序固定（优先级→OrderIndex→CreatedAt）。

### 方案（双行展开型布局）

新增 4 个筛选/排序控件：

- **状态筛选**：RadioButton 组 — 全部 / 未完成 / 已完成，默认"全部"
- **排序字段**：ComboBox — 发布时间 / 完成时间 / 最后修改时间，默认"发布时间"
- **排序方向**：切换按钮，升序 ↑ / 降序 ↓，默认降序
- **等级优先**：CheckBox，勾选后第一级按 Priority desc，同等级内尊重上面选的排序字段+方向

### 改动清单

| 文件 | 改动 |
|---|---|
| `Models/TaskItem.cs` | 新增 `LastModifiedAt` 字段（DateTime，默认 Now） |
| `Services/DatabaseService.cs` | `UpdateTaskAsync` 自动更新 `LastModifiedAt`；`SearchTasksAsync` 扩展参数（status/sortField/sortDesc/priorityFirst），SQLite 层完成排序 |
| `ViewModels/HistoryViewModel.cs` | 新增 `StatusFilter` / `SortField` / `SortDescending` / `PriorityFirst` 属性 |
| `Views/HistoryWindow.xaml` | 关键词行下方加分隔线 + 筛选行（RadioButton）+ 排序行（ComboBox + 方向按钮 + CheckBox） |

### 排序逻辑

```
if (priorityFirst)
    OrderByDescending(Priority)
    → ThenBy(sortField, sortDesc)
else
    OrderBy(sortField, sortDesc)
```

字段映射：发布时间→CreatedAt，完成时间→CompletedAt，最后修改时间→LastModifiedAt。

## 3. 每日自动总结

### 现状

无此功能。

### 方案

**内容形式**：统计 + 简短导语（方案 C）。顶部一句话概括（"今天完成了 X/Y 个任务..."），主体为统计数字 + 已完成/未完成任务清单。

**UI 形式**：独立弹窗 + 自动弹出组合（A + C）：

- `DailySummaryWindow`（约 600×500），顶部日期切换器（◀ 前一天 / 后一天 ▶），底部复制/导出文本按钮
- 主界面底部功能栏新增"每日总结"按钮打开此窗
- 设置中添加开关"跨天时自动弹出昨日总结"，默认关闭

### 数据来源

当日 `TaskItem` 数据，统计：总任务数、完成数、完成率、高优先级完成数，已完成/未完成任务清单（按优先级降序）。

### 改动清单

| 文件 | 改动 |
|---|---|
| `ViewModels/DailySummaryViewModel.cs` | **新建**，日期切换、统计数据计算、复制/导出逻辑 |
| `Views/DailySummaryWindow.xaml/.cs` | **新建**，窗口 UI + 日期切换控件 |
| `ViewModels/MainViewModel.cs` | 新增 `OpenDailySummaryCommand` |
| `Views/MainWindow.xaml` | 底部功能栏加"每日总结"按钮 |
| `Services/SettingsService.cs` | 配置类新增 `AutoShowDailySummary` 属性 |
| `Views/SettingsWindow.xaml` | 设置界面新增自动弹出开关 |
| `ViewModels/MainViewModel.cs` | 跨天检测中，若 `AutoShowDailySummary` 为 true，弹出 DailySummaryWindow（昨日数据） |

## 4. 附加番茄钟删除

### 现状

`RemoveAdditionalTimerCommand` 已在 ViewModel 中实现，但 XAML 中附加组选项卡无删除按钮，只有添加 "+" 按钮。

### 方案

每个附加组选项卡右侧加小 "×" 按钮，点击直接移除。主番茄钟不显示 ×。

### 改动清单

| 文件 | 改动 |
|---|---|
| `Views/MainWindow.xaml` | ListBox ItemTemplate 中，非主番茄钟选项卡右侧加 × 按钮，绑定 `RemoveAdditionalTimerCommand` |

### × 按钮显示条件

通过 `TimerName != "主番茄钟"` 判断，或用 `IsMainTimer` 属性区分。主番茄钟不显示删除按钮。
