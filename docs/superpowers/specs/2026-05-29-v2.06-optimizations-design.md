## 1. 番茄钟 × 删除按钮上下不居中

### 现状

`MainWindow.xaml:513-515`，附加番茄钟选项卡内 × 按钮的 `Padding="2,0"`，无垂直内边距，`FontSize="14"` 的 × 字符在水平 StackPanel 中视觉偏上。

## 2. 每日总结只统计当天创建的任务，遗漏跨创建日期完成的任务

### 现状

`DailySummaryViewModel.LoadSummaryAsync()` 调用 `DatabaseService.SearchTasksAsync(dayStart, dayStart)`，该方法按 `CreatedAt` 过滤。结果是：昨天创建、今天完成的任务不会出现在今天的总结中。

### 方案：混合口径

- **已完成列表**：按 `CompletedAt` 筛选，今天内完成的任务全部纳入（不限创建日期）
- **未完成列表**：按 `CreatedAt` 筛选，今天及之前创建的未完成任务
- **统计数字**：TotalCount = 已完成数 + 未完成数；CompletedCount / CompletionRate / HighPriorityCompleted 仅基于已完成列表
- **SummaryLine 导语**：数据来源替换，文案逻辑不变

### 数据层改动

`DatabaseService` 新增两个查询方法，不改动现有 `SearchTasksAsync`（历史查询等其他调用方不受影响）：

```csharp
// 查询指定日期完成的所有任务
Task<List<TaskItem>> GetTasksCompletedOnAsync(DateTime date)

// 查询指定日期及之前创建且未完成的任务  
Task<List<TaskItem>> GetUncompletedCreatedUpToAsync(DateTime date)
```

`DailySummaryViewModel.LoadSummaryAsync()` 改为分别调用这两个方法后合并统计。

### 改动清单

| 文件 | 改动 |
|---|---|
| `Services/DatabaseService.cs` | 新增 `GetTasksCompletedOnAsync`、`GetUncompletedCreatedUpToAsync` 方法 |
| `ViewModels/DailySummaryViewModel.cs` | `LoadSummaryAsync` 改为双查询合并模式 |

## 3. 历史查询筛选UI风格不一致 + ComboBox 下拉项不可读

### 现状

`HistoryWindow.xaml:246-295` 筛选区存在两个问题：

1. **RadioButton 状态筛选**：WPF 默认 RadioButton 样式，与应用的深色主题 + 紫调霓虹风格完全不搭
2. **ComboBox 排序字段**：只有 `Background="#1A6366F1"` 和 `Foreground`，未自定义下拉 Popup 模板，导致下拉项使用系统默认白底 + 继承浅色字体，背景与字体色几乎一致，用户看不清选项

### 方案

**状态筛选 Chip 按钮组**：

RadioButton 组替换为互斥 ToggleButton 组，使用与主窗口一致的胶囊按钮样式（`SecondaryButton` 变体），选中态高亮为主题紫 `#6366F1`。底层绑定改为 ViewModel `StatusFilter` 属性，去 `StatusFilter_Click` 事件。

**排序 ComboBox 深色模板**：

给 ComboBox 添加完整 `ControlTemplate`：
- 下拉 Popup 内 `Border` 背景 `#252540`，`BorderBrush="#666366F1"`
- `ComboBoxItem` 统一 `Foreground="{StaticResource TextPrimaryBrush}"`，选中高亮 `#6366F1`
- 下拉箭头用 Path 绘制，匹配主题风格

**视觉微调**：

- 日期 `DropShadowEffect` Opacity 从 0.2 → 0.35
- 整体筛选区间距对齐主窗口组件风格

### 改动清单

| 文件 | 改动 |
|---|---|
| `Views/HistoryWindow.xaml` | RadioButton → Chip ToggleButton 组；ComboBox 完整深色模板；日期阴影微调 |
| `Views/HistoryWindow.xaml.cs` | 移除 `StatusFilter_Click` 事件处理 |
| `ViewModels/HistoryViewModel.cs` | 新增 `StatusFilter` 绑定属性（替换事件绑定） |
