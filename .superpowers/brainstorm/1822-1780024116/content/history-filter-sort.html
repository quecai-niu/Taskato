HistoryWindow 筛选排序区 — 两种布局方案

新增控件：状态筛选 + 排序字段 + 升/降序 + 等级优先勾选。请选择你觉得顺眼的布局

A

单行紧凑型

所有筛选排序控件放在关键词行下方，与搜索结果计数同行，节省垂直空间

┌──────────────────────────────────────────────┐
│ 开始日期 [____] 至 [____] │
│ 关键词 [____________] [🔍搜索] [⬇导出] │
│ 状态:[全部▾] 排序:[发布时间▾] [↑] ☑等级优先 │ ← 新行
│ 找到 12 条记录 │
├──────────────────────────────────────────────┤
│ (任务列表...) │
└──────────────────────────────────────────────┘
B

双行展开型

搜索按钮移到最右侧，筛选排序独占一行，视觉分组更清晰，适合小屏或密集操作

┌──────────────────────────────────────────────┐
│ 开始日期 [____] 至 [____] │
│ 关键词 [____________] [🔍搜索] [⬇导出] │
│──────────────────────────────────────────────│ ← 分隔线
│ 筛选: ○全部 ○未完成 ○已完成 │
│ 排序: [发布时间 ▾] 方向:[↑升序] ☑ 等级优先 │
│ 找到 12 条记录 │
├──────────────────────────────────────────────┤
│ (任务列表...) │
└──────────────────────────────────────────────┘

控件说明

状态筛选 — 三个 RadioButton：全部 / 未完成 / 已完成，默认"全部"

排序字段 — ComboBox：发布时间 / 完成时间 / 最后修改时间，默认"发布时间"

排序方向 — 切换按钮，点击在升序 ↑ / 降序 ↓ 之间切换，默认降序

等级优先 — CheckBox，勾选后第一级按优先级降序，同等级内部尊重上面选的排序字段+方向

配套改动

1. TaskItem 新增 LastModifiedAt 字段 — 每次 UpdateTaskAsync 自动更新

2. SearchTasksAsync 签名扩展 — 接收 status/sortField/sortDesc/priorityFirst 参数，在 SQLite 层完成排序（非内存排序）

3. 排序字段映射：发布时间→CreatedAt，完成时间→CompletedAt，最后修改时间→LastModifiedAt
