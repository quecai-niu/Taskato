每日总结 UI — 三种呈现形式

同一种内容（方案C的统计+导语），不同呈现方式

A

独立弹窗（推荐）

类似历史查询窗，从主界面按钮打开，顶部有日期切换器可翻看历史总结

B

主界面内嵌面板

在主窗口底部加可折叠区域，打开就看到今日小结，无需额外窗口，但会挤占任务列表空间

自动弹出

跨天或首次打开时自动弹窗展示昨日/今日总结，可设"不再提醒"

我的推荐

推荐 A + C 组合：A 作为常设功能（主界面有按钮打开独立窗口，可翻看任意日期），C 作为可选开关（设置里勾选"跨天时自动弹出昨日总结"）。

B 方案会挤占主界面本就不大的任务列表区域，且主窗口职责已经很重，不推荐。
